ROUTE 02 / OPERATING PROTOCOL

Portable by design.

Knowledge travels with the skill. The operator keeps control.

  1. 01

    Discover

    Compatible agents read lightweight metadata and recognize when a skill applies.

  2. 02

    Load

    Detailed instructions and resources are loaded only when they are relevant.

  3. 03

    Operate

    The agent follows a documented workflow, uses local tools, and surfaces prerequisites.

  4. 04

    Verify

    Tests, references, caveats, and human confirmation protect consequential actions.

ROUTE 03 / TRUST BOUNDARY

Convenience must remain inspectable.

Agent-ready does not mean agent-unaccountable. Skills document the edges.

VISIBLE

Instructions are source.

Workflows live in readable files. Review what an agent will do before it does it.

SCOPED

Context loads on demand.

Agents discover compact metadata first and load operational detail only when needed.

HONEST

Caveats travel with capability.

Good skills name prerequisites, trust assumptions, failure modes, and confirmation points.

TESTABLE

Claims can be checked.

Scripts, examples, references, and tests make workflows easier to audit and improve.

Operator checklist / before install

  1. Read SKILL.md
  2. Inspect bundled scripts
  3. Confirm network and account scope
  4. Review safety assumptions
  5. Run available tests or security scans
  6. Keep a human in consequential loops
