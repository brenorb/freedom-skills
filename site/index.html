Why we gather

Convenience can belong to everyone

The best tools for autonomy shouldn’t be the hardest to use.

Bitcoin, Nostr, privacy tools, and other freedom technologies can help people communicate, transact, preserve information, and coordinate. But each tool brings a new interface, vocabulary, setup process, and set of security assumptions. Freedom Skills packages that knowledge so compatible agents can help developers, activists, researchers, and human-rights defenders act with greater confidence.

Pack once. Travel widely.

A skill is a portable piece of know-how.

At minimum, each skill contains a SKILL.md file. It may also bring scripts, references, templates, and tests. Compatible agents read lightweight metadata, recognize when a skill applies, and load its detailed workflow only when needed.

Explore the open Agent Skills standard ↗
  1. 01
    Discover

    The agent notices that a task matches a skill’s purpose.

  2. 02
    Load

    It brings in the relevant instructions and resources—not the whole library.

  3. 03
    Act with context

    It follows a tested workflow and surfaces prerequisites, choices, and caveats.

  4. 04
    Stay portable

    The same open package can travel across compatible agents and tools.

Trust is part of the workflow

Guidance, not a magic wand.

Freedom Skills makes powerful tools more approachable without pretending risk disappears. Good skills explain prerequisites, security assumptions, failure modes, and moments that need human confirmation. Before installing, inspect the source. Before sensitive action, understand what leaves your machine. Keep keys and final judgment with people.

  • ✓ Open, inspectable instructions
  • ✓ Explicit safety assumptions
  • ✓ Security scanning before install
  • ✓ Human confirmation for consequential action
