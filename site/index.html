PACK ONCE · CARRY WIDELY

A skill is a small press kit for an agent.

At minimum, it contains a SKILL.md. It can also carry scripts, references, templates, and other resources. Compatible agents read the metadata, recognize when the skill applies, then load the detailed workflow only when needed.

Built on the open Agent Skills standard ↗

  1. 1
    Discover

    The agent sees a compact name and task description.

  2. 2
    Load

    Detailed instructions arrive only when the task matches.

  3. 3
    Act with context

    Scripts and references support a repeatable workflow.

  4. 4
    Move the folder

    The package travels between compatible agents without rewriting the know-how.

READ
BEFORE
RUNNING

Convenient
doesn’t mean
careless.

Each workflow should surface prerequisites, trust assumptions, and irreversible steps—not hide them behind a friendly command.

  • Inspect first. Skills are plain-text instructions. Read them and scan third-party skills before installing.
  • Keep people in the loop. Posting, sending, paying, deleting, and publishing deserve explicit intent.
  • Protect secrets. Private keys, mnemonics, credentials, contact data, and sensitive files require careful handling.
  • Know the boundary. Peer-to-peer, hosted, and archival services carry different metadata and availability tradeoffs.

Also in the repo: a portable Rampart pre-send / post-send hook pair for redacting selected personal information before model requests and restoring placeholders in replies.
